{{ _('How do I connect my Kobo?') }}
  1. {{ _('Generate a URL above and copy it.') }}
  2. {{ _('Connect your Kobo to a computer via USB. A KOBOeReader drive appears.') }}
  3. {{ _('On that drive, open this exact file in a text editor:') }}
    .kobo/Kobo/Kobo eReader.conf
    {{ _('Note the leading dot — the .kobo folder is hidden on most operating systems. The file lives a few levels deep; do not edit any other .conf you might find. Optional but recommended: copy the file to Kobo eReader.conf.bak first as a backup.') }}
  4. {{ _('Find the [OneStoreServices] section. Replace the existing api_endpoint= line with:') }}
    api_endpoint={{ host_url }}/kobo/<your-url-key>
  5. {{ _('In the same [OneStoreServices] section, delete these three lines if they exist:') }}
    image_host=
    image_url_template=
    image_url_quality_template=
    {{ _('Stale values here (often left over from a previous Komga or Calibre-Web setup) silently override the ones Colophon sends, which is the #1 cause of "books sync but covers are blank". Deleting lets Colophon supply fresh ones on the next sync.') }}
  6. {{ _('Save the file. Eject the Kobo safely — wait a few seconds first so the device finishes flushing the write.') }}
  7. {{ _('On the Kobo: Settings → Sync now. Books from Colophon appear in your Library.') }}
    {{ _('First sync of a large library can take a few minutes — covers and books arrive in batches. Keep the device awake until the Devices table above shows the sync count incrementing.') }}

{{ _('To undo: restore api_endpoint=https://storeapi.kobo.com in the .conf file and sync again. The device returns to the official Kobo store.') }}

  • {{ _('Reset reading state on a book → clears it back to Unread everywhere, useful if a sync got stuck or you want to re-read.') }}

  • {{ _('Pair a second Kobo and it inherits all reading state on first sync — no per-device merge step.') }}

{{ _('Troubleshooting') }}

{{ _('Sync runs but no books appear on the Kobo.') }}
{{ _('Confirm the Devices table above shows the sync count incrementing after you press Sync now. If it does, the Kobo received data but is filtering it — check that the books in Colophon are EPUB or KEPUB. If the count does not increment, the .conf api_endpoint is wrong or the device cannot reach Colophon on the network.') }}

{{ _('Books appear but covers stay blank.') }}
{{ _('This is almost always a wrong base URL in the configuration: the Kobo strips the port from the Host header on some firmware, so cover URLs end up pointing at port 80. Make sure the COLOPHON_PUBLIC_URL environment variable on the server includes the port (e.g. http://192.168.1.100:5055). Restart Colophon, then on the Kobo: edit the .conf, delete any image_host/image_url_template lines, save, sync.') }}

{{ _('The Kobo was previously paired with Komga or Calibre-Web.') }}
{{ _('Old entitlements live in the device’s KoboReader.sqlite. They appear alongside Colophon books and will 404 when opened. To fully clean up: connect via USB, delete .kobo/KoboReader.sqlite, and let the device run its first-boot setup again (you will need to re-sign in to a Kobo account). Less drastic: ignore the broken entries.') }}

{{ _('Reading progress from the Kobo never updates in Colophon.') }}
{{ _('Make sure you press Sync now (or close the book) on the Kobo — state is pushed in sync windows, not on every page turn. If still nothing arrives, check docker logs for a line like "Kobo state PUT: book_id=..." — its absence means the device is not sending state at all (rare; usually a stuck local cache that resolves after a device restart).') }}
